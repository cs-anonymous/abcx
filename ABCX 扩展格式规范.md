# ABCX — 扩展 ABC 乐谱格式规范

> **ABCX**（Extended ABC Notation）是基于 ABC 标准 v2.1 的扩展格式，旨在为 LLM Agent 作曲系统和多声部钢琴乐谱提供更简洁、更直观的文本表示。
>
> **设计原则**：以作曲者思维为中心，保留换行信息，消除声部对齐负担，支持任意数量声部。
>
> **文件后缀**：`.abcx`

---

## 一、与 ABC 标准的关系

| 维度 | ABC 标准 v2.1 | ABCX 扩展 |
|------|--------------|----------|
| 文件后缀 | `.abc` | `.abcx` |
| 头部 | 兼容 | 兼容 + `%%score` 分组声明 |
| 曲体 — 单声部 | 兼容 | 兼容 |
| 曲体 — 多声部 | `V:` 字段 + 换行 + `&` 多轨 | `;` 内联声部分隔 + inline `&` |

**核心差异**：
- ABC 标准的多声部需要跨行声明（`V:1`、`V:2`…），LLM 生成时需要精确对齐时值
- ABCX 的多声部在同一小节内用 `;` 分隔，每个声部独立书写，无需对齐
- ABCX 将谱表归属从曲体分离到头部 `%%score` 声明

---

## 二、头部（Header）

ABCX 头部完全兼容 ABC 标准 v2.1 所有字段。

### 2.1 `%%score` 声部-谱表分组声明

**语法**：`%%score (voice_list) (voice_list) ...`

每个括号组对应一个谱表，组内 voice 共享该谱表，用花括号连接。

```abcx
%%score (V1) (V2)            // 钢琴 2 声部
%%score (V1 V2) (V3 V4)      // 钢琴 4 声部赋格
%%score (V1 V2) (V3)         // 上行 2 声部 + 下行 1 声部
```

**Voice 命名**：`V` + 数字（`V1`、`V2`…），大小写不敏感。曲体中小节内 `;` 分隔数 + 1 必须等于 voice 总数。如果声部没有音则该位置留空

> `%%score` 在 ABC 标准中是可选排版指令，ABCX 中升级为**必需的结构声明**。

### 2.2 可选的声部属性

```abcx
V:1 name="Soprano" clef=treble
V:2 name="Alto" clef=treble
V:3 name="Tenor" clef=bass
V:4 name="Bass" clef=bass
```

省略时默认谱号：第一组 → treble，第二组 → bass。

## 三、曲体（Body）— 核心扩展

### 3.1 `;` 声部分隔符

一个小节内，`;` 出现次数 = 声部数 − 1。第 n 个 `;` 之前的内容属于 Vn。

```
| <V1内容> ; <V2内容> ; <V3内容> ... |
```

**示例（2 声部钢琴）**：

```abcx
%%score (V1) (V2)

| c d e f g a b c ; C2 G2 E2 C2 |
```

- V1（高音谱表）：`c d e f g a b c`（8 个八分音符）
- V2（低音谱表）：`C2 G2 E2 C2`（4 个四分音符）

**关键**：两个声部不需要逐音对齐，但每个声部在一个小节内的总时值必须与该小节拍号匹配。休止必须显式写出，不能由解析器自动补齐。

**示例（4 声部赋格）**：

```abcx
%%score (V1 V2) (V3 V4)

| c2 d2 e2 f2 ; e2 f2 g2 a2 ; G2 A2 B2 c2 ; C,4 |
```

### 3.2 `&` 同一声部内的多轨

`&` 用于分隔同一 voice 内节奏独立的多条线（如旋律 + 内声部），不要求换行。

`&` 的优先级低于 `;`——`&` 只在单个 voice 内部生效。

```
| <track_A> & <track_B> ; <voice_2> |
```

**示例（右手旋律 + 内声部）**：

```abcx
M:4/4
L:1/8
%%score (V1) (V2)

| c d e f g a b c & z4 g2 a2 ; C2 G2 E2 C2 |
```

- V1 有两层：旋律 `c d e f g a b c`（8 个八分音符） + 内声部 `z4 g2 a2`（4 个八分休止 + 2 个四分音符）
- V2：`C2 G2 E2 C2`

**时间规则**：`&` 分隔出的每一层都是从当前小节起点开始的独立时间线。每一层在严格模式下都必须填满当前小节；如果某层没有发声，必须显式写休止符。

**与 `[ ]` 和弦的区别**：

| 语法 | 含义 |
|------|------|
| `[CEG]` | 所有音**同时发声**（和弦） |
| `c e g &` | 多轨**节奏独立**，时间上不完全重叠 |

### 3.3 `&` 和 `;` 的组合

```abcx
M:4/4
L:1/8
%%score (V1 V2) (V3 V4)

| c d e f g a b c & z4 g2 a2 ; e f g a b c' d' e' & z4 b2 c'2 ; C2 G2 E2 C2 ; C,4 G,,4 |
```

解析顺序：先用 `;` 分割为 4 个 voice，再对每个 voice 内部检查 `&` 拆分为多轨。

结果：
- V1：`c d e f g a b c` + `z4 g2 a2`（两轨）
- V2：`e f g a b c' d' e'` + `z4 b2 c'2`（两轨）
- V3：`C2 G2 E2 C2`（单轨）
- V4：`C,4 G,,4`（单轨）

### 3.4 时间完整性规则

ABCX 采用严格时间模型：

1. `;` 分隔不同 voice。
2. `&` 分隔同一 voice 内的多个 layer。
3. 每个 voice/layer 在每个小节内都必须形成完整时间线。

所谓“完整时间线”，是指该 voice/layer 内所有音符、和弦、休止符、连音符等时值之和，必须等于当前小节的目标时值。

```abcx
M:4/4
L:1/8
%%score (V1) (V2)

| c d e f g a b c ; C2 G2 E2 C2 |   // 合法：两个 voice 都是 8 个八分音符
| c d e z z z z z ; C2 G2 E2 C2 |   // 合法：V1 显式补足休止
| c d e ; C2 G2 E2 C2 |             // 非法：V1 只有 3 个八分音符
```

多 layer 时，每个 layer 也必须独立完整：

```abcx
| c d e f g a b c & z4 g2 a2 ; C2 G2 E2 C2 |  // 合法
| c d e f & g2 a2 ; C2 G2 E2 C2 |              // 非法：两个 layer 都未填满小节
```

如果某个 voice 在某小节完全没有内容，也必须写出完整休止：

```abcx
| c d e f g a b c ; z8 |
```

解析器不得自动补休止符。时值不完整属于格式错误。

### 3.5 保留的换行语义

换行具有语义——每行对应乐谱渲染时的一行谱表。相邻小节共用 `|`（继承 ABC 标准写法）：

```abcx
c d e f g a b c ; C2 G2 E2 C2 |
e2 d2 c4 B4 ; A,2 B,2 C2 D2 |
A4 B4 c4 d4 ; C,4 G,,4 |
```

保留"一行 = 一个时间切片"的思维模式。

**注意**：曲子内部**不允许空行**（空行表示曲子结束）。

### 3.6 曲内速度变化

ABCX 支持在曲体中用 inline `[Q:]` 改变速度：

```abcx
[Q:1/4=60]           ← 速度变为 ♩=60
[Q:rit.]            ← 渐慢（谱面标记，具体时值由渲染器处理）
[Q:a tempo]         ← 恢复原速
```

**示例**：

```abcx
M:4/4
L:1/8
|: "Eb" !p! g2 a2 b2 c'2 ; "Eb" Eb,2 B,2 G,2 B,2 |
   "Bb" d'2 c'2 b2 a2 ; "Bb" Bb,2 F2 D2 F2 |
   [Q:rit.] "Ab" e2 d2 c2 d2 ; "Ab" Ab,2 Eb2 C2 Eb2 |
   [Q:a tempo] "Bb" f2 g2 a2 b2 ; "Bb" Bb,2 F2 D2 F2 :|
```

### 3.7 曲内拍号与默认长度变化

ABCX 允许在曲内切换 `M:` 和 `L:`，但切换必须对所有 voice / staff 同步，不能出现不同声部使用不同默认长度的情况。

**规则**：
- `[M:]` 和 `[L:]` 出现在独立行时，对其后所有小节生效，直到下一次被修改。
- `[M:]` 和 `[L:]` 也可以写在某个小节的第一声部内容中，从该小节开始生效。
- 同一时刻所有 voice / staff 必须共享相同的有效 `M:` / `L:`；转换器应保留这些同步切换，而不是把它们压扁成单值。
- 对于生成数据集，推荐尽量减少 `L:` 变化，但如果音乐结构需要，允许在乐句或段落边界切换。

```abcx
M:4/4
L:1/8
%%score (V1) (V2)

| c d e f g a b c ; C2 G2 E2 C2 |    // 4/4 拍
| g a b c d e f g ; G,2 D2 B,2 G,2 |
[M:3/4]
| c d e f g a ; C2 G2 E2 |             // 3/4 拍
| g a b c d e ; G,2 D2 B,2 |
[M:2/4] [L:1/16]
| cdef gabc ; C2G2E2C2 |              // 2/4 拍，十六分音符
```

也可以 inline 写在小节内容中：

```abcx
| c d e f g a b c ; C2 G2 E2 C2 |
| [M:3/4] c d e ; g a b |             // 从该小节起变为 3/4
| c d e ; g a b |                     // 后续小节仍为 3/4
```

注：`[L:]` 示例表示标准化后的 ABCX 允许保留这种同步切换；关键是同一位置不能出现声部间不一致。

### 3.8 自由文字标注

用 `"^text"`（音符上方）和 `"vtext"`（音符下方）在五线谱上放置任意演奏术语或注释：

```abcx
| "^dolce" c4 d4 e4 f4 ; C,4 |
| "^cantabile" g4 a4 b4 c'4 ; "vsempre legato" Ab,2 Eb2 |
```

**常见标记**：`dolce`、`cantabile`、`sempre legato`、`con fuoco`、`espressivo`、`leggiero`、`marcato` 等。

### 3.9 范围型标记

范围型标记是具有明确起点和终点的演奏/排版语义，例如渐强、渐弱、踏板、八度线、长颤音、乐句线等。

ABCX 支持两种范围型标记写法：

1. 继承 ABC 原生的成对标记，例如 `!crescendo(! ... !crescendo)!`。
2. ABCX 扩展的显式作用域标记，用于多 voice / 多 layer 场景。

#### 3.9.1 默认绑定规则

范围型标记默认绑定到它所在的 layer。

```abcx
| !crescendo(! c d e f g a b c !crescendo)! ; C2 G2 E2 C2 |
```

上例中渐强只作用于 V1 的当前 layer，不作用于 V2。

如果范围标记写在某个 `&` layer 中，则只作用于该 layer：

```abcx
| c d e f g a b c & !crescendo(! z4 g2 a2 !crescendo)! ; C2 G2 E2 C2 |
```

上例中渐强只作用于 V1 的第二个 layer。

#### 3.9.2 显式作用域标记

当范围需要作用于整个 voice、整个谱表，或多个 voice/layer 时，使用 ABCX 显式作用域标记。

**语法**：

```abcx
@[scope:id:kind(
@[scope:id:kind)
```

- `scope`：作用域
- `id`：范围标记 ID，用于配对起止
- `kind`：标记类型
- `(`：范围开始
- `)`：范围结束

**作用域格式**：

| 作用域 | 含义 |
|--------|------|
| `V1` | V1 的所有 layer |
| `V1.1` | V1 的第 1 个 layer |
| `V1.2` | V1 的第 2 个 layer |
| `S1` | 第 1 个谱表内所有 voice/layer |
| `ALL` | 全谱所有 voice/layer |

**示例：作用于整个右手 voice**

```abcx
| @[V1:cr1:crescendo( c d e f g a b c & z4 g2 a2 ; C2 G2 E2 C2 |
| g a b c' d' e' f' g' & z4 b2 c'2 @[V1:cr1:crescendo) ; G,2 D2 B,2 G,2 |
```

**示例：作用于整个第一谱表**

```abcx
| @[S1:ot1:ottava8va( c' d' e' f' g' a' b' c'' ; e' f' g' a' b' c'' d'' e'' ; C2 G2 E2 C2 |
| g' a' b' c'' d'' e'' f'' g'' ; b' c'' d'' e'' f'' g'' a'' b'' @[S1:ot1:ottava8va) ; G,2 D2 B,2 G,2 |
```

#### 3.9.3 配对规则

显式作用域范围标记必须满足：

1. 开始与结束的 `scope`、`id`、`kind` 必须完全一致。
2. 同一 `scope` 内，同一 `id` 不得嵌套复用。
3. 范围可以跨小节、跨换行，但不得跨曲目结束。
4. 范围标记不占时值。
5. 解析器必须校验未闭合、未开始、重复 ID 等错误。

常见 `kind` 建议值：

| kind | 含义 |
|------|------|
| `crescendo` | 渐强 |
| `diminuendo` | 渐弱 |
| `pedal` | 踏板 |
| `slur` | 圆滑线/乐句线 |
| `phrase` | 乐句 |
| `ottava8va` | 高八度 |
| `ottava8vb` | 低八度 |
| `trill` | 长颤音 |

普通单 layer 场景优先使用 ABC 原生标记；只有当作用域可能产生歧义时，才需要使用 `@[scope:id:kind(...)]`。

### 3.10 默认渲染策略

ABCX 不要求作者手动指定符干方向、碰撞避让等刻谱细节。渲染器应采用默认渲染策略：

1. 同一谱表两个 voice 时，上方 voice 默认符干向上，下方 voice 默认符干向下。
2. 同一 voice 内多个 layer 时，奇数 layer 默认符干向上，偶数 layer 默认符干向下。
3. 同一谱表超过两个 voice/layer 时，由渲染器自动分配符干方向与避让。
4. 手动刻谱控制不属于 ABCX 0.1 的核心目标，可在后续版本扩展。

### 3.11 完整示例

**示例 1：2 声部钢琴（夜曲片段）**

```abcx
X:1
T:夜曲片段
M:4/4
L:1/8
K:Cm
%%score (V1) (V2)

|: "Cm" !pp! (c2 d2) e2 f2 ; "Cm" C,4 G,,4 |
   "Ab" g2 a2 b2 c'2 ; "Ab" Ab2 Eb2 C2 Ab2 |
   "Bb" d'2 c'2 b2 a2 ; "Bb" Bb2 F2 D2 Bb2 |
   "Gm" g2 f2 e2 d2 ; "Gm" G2 D2 B,2 G,2 :|
```

**示例 2：4 声部赋格（巴赫风格）**

```abcx
X:2
T:赋格片段
M:4/4
L:1/4
K:C
%%score (V1 V2) (V3 V4)
V:1 name="Soprano" clef=treble
V:2 name="Alto" clef=treble
V:3 name="Tenor" clef=treble8
V:4 name="Bass" clef=bass

| c d e f ; e f g a ; G A B c ; C, D, E, F, |
  g a b c' ; b c' d' e' ; d e f g ; G, A, B, C, |
  [ce] [df] [eg] [fa] ; [eg] [fa] [gb] [ac'] ; [Bd] [ce] [df] [eg] ; [E,G,] [F,A,] [G,B,] [A,C] |]
```

**示例 3：带内声部的夜曲（右手两轨 + 左手）**

```abcx
X:3
T:夜曲（带内声部）
M:3/4
L:1/8
K:Eb
%%score (V1) (V2)

|: "Eb" !p! (g a b) g f e & z4 c2 ; "Eb" Eb,2 B,2 G,2 |
   "Bb" (f g a) f e d & z4 e2 ; "Bb" Bb,2 F2 D2 |
   "Ab" (e f g) e d c & z4 d2 ; "Ab" Ab,2 Eb2 C2 :|
```

## 四、继承自 ABC 标准的 Score 层特性

以下特性完全继承 ABC 标准 v2.1 语法，ABCX 不做任何修改：

| 类别 | 语法 | 说明 |
|------|------|------|
| **音符与八度** | `A-G a-g , '` | 音名、大小写区分八度、逗号/撇号扩展 |
| **升降还原** | `^ ^^ _ __ =` | 升号、重升、降号、重降、还原 |
| **时值修饰** | `2 3 / 3/2` | 倍长、三倍长、减半、附点等 |
| **节奏快捷** | `> <` | 前长后短 / 前短后长 |
| **休止符** | `z z2 z/` | 时值规则同音符 |
| **调号** | `K:` | 调式、调号、模态（`dor mix lyd` 等） |
| **拍号** | `M:`（头部）/ `[M:]`（曲体 inline） | `4/4 3/4 6/8 C C|` |
| **速度** | `Q:`（头部） | `Q:1/4=72` |
| **小节线** | `\|\| \|] \|\| \|: :\| :: \|1 \|2` | 反复、不同结尾 |
| **连音线/圆滑线** | `- ()` | Tie 连接同音、Slur 连奏 |
| **连音符** | `(3GAB (5:4:cdef` | 三连音、五连音等 |
| **和弦** | `[CEGc]` | 方括号内同时发声 |
| **装饰音** | `~ {g}a` | 转音、精确倚音 |
| **伴奏和弦** | `"Gm7" "D7/F#"` | 双引号和弦标记 |
| **力度/演奏标记** | `!pp! !f! !ped! !crescendo(! .g Td Hd Md` | 力度、踏板、渐强/弱、断奏、颤音、延长号 |
| **段落标记** | `P:A P:B`（头部/曲体） | 段落标注与演奏顺序 |
| **注释** | `%` | 行内注释，之后内容被忽略 |
| **歌词对齐** | `w:` / `W:` | 逐音符对齐歌词 / 整段歌词 |
| **续行符** | `\` | 行尾续行，属于同一谱表行 |

## 五、ABCX → 标准 ABC 转换

ABCX 可无损转换为标准 ABC v2.1 以便 abcjs/abcm2ps 渲染。

**转换规则**：

| ABCX 语法 | 标准 ABC 等价 |
|-----------|--------------|
| `;` 声部分隔 | 拆分为 `[V:n]` 行 |
| inline `&` | 拆分为换行 `&`（需校验每个 layer 的时值完整性） |
| 换行 | 保留为行分组 |

**转换示例**：

```
ABCX:  | c d e f g a b c & z4 g2 a2 ; C2 G2 E2 C2 |
ABC:   [V:1] c d e f g a b c
       &
       z4 g2 a2
       [V:2] C2 G2 E2 C2
```

## 六、快速参考

### 6.1 ABCX 扩展语法

| 符号 | 含义 |
|------|------|
| `;` | **声部分隔符** |
| `&`（inline） | **同 voice 内多轨** |
| `%%score` | **声部-谱表分组**（必需声明） |

### 6.2 声部优先级

三层嵌套，从低到高：
1. `;` 分隔不同 voice
2. `&` 分隔同一 voice 内多轨
3. `[ ]` 同时发声的和弦

### 6.3 头部字段

| 字段 | 必需 | 说明 |
|------|------|------|
| `X:` | ✅ | 索引编号 |
| `T:` | ✅ | 标题 |
| `M:` | ✅ | 拍号 |
| `L:` | ✅ | 默认音符长度 |
| `K:` | ✅ | 调号（必须最后） |
| `%%score` | ✅ ABCX | 声部-谱表分组 |
| `V:n` | 可选 | 声部属性（谱号、名称） |
| 其余 ABC 字段 | | 兼容标准 |
